# Devlog 2026-03-06 - Auto Incremental Sync on USB Plug-In

## Scope
Implemented USB plug-in triggered auto-sync with persistent incremental state, idempotency guardrails, lock/debounce concurrency protection, and pluggable notes storage interface scaffold.

## What Changed
- Added persistent sync state store:
  - `src/syncState.ts`
  - stores `lastSuccessfulSyncAt`, `lastRunStartedAt`, and processed file markers
- Added sync run coordinator with debounce + single-flight behavior:
  - `src/syncCoordinator.ts`
- Added pluggable notes storage interface + local adapter:
  - `src/notesStorage.ts`
  - TODO hook for future memdock adapter
- Updated workflow to consume storage adapter:
  - `src/meetingWorkflow.ts`
- Refactored meetings sync CLI into reusable runner:
  - `src/cli/meetingsSync.ts`
  - exported `runMeetingsSync` and `parseArgs`
  - preserved existing manual CLI behavior
  - added `--state-file` support
  - guarded entrypoint with `isDirectRun`
- Wired USB watcher to auto-run sync on plug-in:
  - `src/cli/usbWatch.ts`
  - added `--auto-sync/--no-auto-sync`
  - added `--sync-debounce-ms`
  - runs sync through coordinator on each plug-in event
  - logs sync start/end and counts from sync runner
- Public exports:
  - `src/index.ts`
- Documentation updates:
  - `README.md`
  - `tasks/todo.md`
  - `tasks/lessons.md`
