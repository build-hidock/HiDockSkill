### 2. ASR switch: Whisper → Moonshine, with auto language detection

**Motivation:** asrbench had recommended Whisper, but recent web research showed
Moonshine has expanded to 8 languages (en, es, ar, ja, ko, vi, uk, zh) via the
"Flavors of Moonshine" release. Moonshine is ~13× smaller than Whisper-turbo,
runs on CPU at ~21× real-time on M-series Macs, and frees Metal for the LLM.
User confirmed personal/non-commercial use → license restriction on non-English
models is moot.

**Changes:**
- `src/transcribe.ts:77-81` — flipped default `ASR_BACKEND` from `dicow` to
  `moonshine`. `dicow` retained as opt-in fallback.
- `scripts/moonshine_transcribe.py` — full rewrite:
  1. Resolve target language: env `LANGUAGE_HINT` > CLI arg > faster-whisper tiny LID
  2. Normalize codes (`en-US`→`en`, `cmn`→`zh`, etc.)
  3. Fall back to English if detected language is outside the 8-language Moonshine set
  4. **Pin `ModelArch.BASE`** explicitly — `get_model_for_language()` defaults to
     `MEDIUM_STREAMING` for English, which is tuned for low-latency live transcription
     and runs ~9× slower for offline use (130s vs 14.8s on the same 5min audio)
  5. Preserve the existing JSON output schema, add `detected_language` and `model_arch`
     fields (parser ignores extras gracefully — no TS interface change needed)
- Diarization is fully preserved: `Transcriber(model_path, model_arch)` with no
  options enables moonshine_voice's built-in speaker_index/has_speaker_id pipeline.

## Key Findings / Lessons

1. **`moonshine_voice.get_model_for_language("en")` defaults to `MEDIUM_STREAMING`**,
   not the BASE model that asrbench measured. For offline batch use, always pass
   `ModelArch.BASE` explicitly. The streaming variants are tuned for live latency,
   not throughput, and they're ~9× slower for offline.
2. **Network instability on `download.moonshine.ai`** caused the first BASE download
   to fail mid-stream (104MB decoder). The package leaves a `.partial` file behind that
   blocks retry — had to delete it manually. Worth surfacing this in the script with a
   helpful error message in a follow-up.
3. **`embeddinggemma-300m` is for IntentRecognizer**, not speaker embeddings. The
   moonshine_voice diarization is opaque from Python — `speaker_index` is exposed but
   no embedding vector behind it. Speaker enrollment will need a parallel ECAPA-TDNN pass.
4. **Diarization works for non-English models too** — the `Transcriber` API is uniform
   across languages, but I haven't yet tested a non-English file. Documented as a
   follow-up in `tasks/todo.md`.
