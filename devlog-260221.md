## Discoveries

- File names from device list include a trailing NUL byte and must be normalized for display.
- `0x0005` returns large chunks (observed `8180` bytes per frame body).
- Observed recordings on current device report file version `0` and stream as MP3 frames (`0xff 0xf3 ...`).

## Mistakes / Fixes

- Initial `0x0015` range-read probe left a pending request during timeout handling.
- Switched validation to stable `0x0005` chunk transfer flow and kept parser/transport logic centered on confirmed command behavior.
- Updated live probe read logic to avoid timeout race conditions that can leave pending USB requests on close.
