# Dev Log - 2026-03-12

## Summary
Added HiDock H1 support, simplified source categories, and improved galaxy dashboard reliability.

## What changed

### 1. HiDock H1 file list protocol support
- **Root cause**: H1 uses a different USB file list format than P1. P1 sends `FF FF` marker + U32BE file count header before file entries. H1 sends raw file entries with no marker and no count.
- **Fix**: `parseHiDockFileListBody()` now detects the presence of `FF FF` marker. Without it, entries are parsed until the body runs out.
- **Verified**: H1 recording `200011-185743-Rec00.hda` (1.2MB) successfully downloaded, transcribed, and saved.

### 2. Galaxy dashboard categories simplified
- Consolidated 5 source types (Meeting, WIP, Room, Call, Whisper) into 3 user-facing categories:
  - **Call** (green) — rec, call
  - **Whisper** (blue) — whsp, wip
  - **Room** (amber) — room
- Legend updated to show only the 3 categories.

### 3. Galaxy server stability fix
- `openGalaxySyncing()` no longer restarts the server if one is already running. Repeated USB plug-in events (common with noisy USB detection) no longer wipe loaded graph data.
- Fixed EADDRINUSE handling when standalone galaxy server is already bound to port 18180.

### 4. Sync progress and incremental state (from prior session)
- Per-file sync progress displayed in web overlay during sync.
- Sync state saved incrementally per file via `markFileProcessed()` so interrupted syncs resume correctly.
- New notes highlighted in list view with purple glow animation.
- New synced cards 30% bigger in galaxy view.

## Verification
- H1 file parsed correctly: 1 file, 57 bytes, no marker format.
- P1 files continue to work (98 files with FF FF marker).
- Build passes, watcher restarts cleanly via launchd.
